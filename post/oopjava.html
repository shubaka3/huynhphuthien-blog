OOP trong Java: Học qua ví dụ thực tế

Hiểu về Lập trình Hướng đối tượng thông qua các ví dụ dễ hiểu

Khái niệm OOP trong Java và Ví dụ (Phần 1)

Từ lâu Java nổi tiếng với việc hướng các lập trình viên phát triển suy nghĩ như các đối tượng thực tế, nó bao gồm dữ liệu và hành vi trong các khối được tạo ra có thể tái sử dụng. Từ các ứng dụng nhỏ đến hệ thống lớn, OOP trong Java luôn là định hướng không thể thiếu. Nó mở đường cho các giải pháp phần mềm có tính modular, đồng thời đi kèm với đó là khả năng mở rộng và sự mạnh mẽ vượt trội so với lập trình thủ tục truyền thống.

Sau đây, chúng ta sẽ cùng nhau đi tìm hiểu về các nguyên tắc, khái niệm và kỹ thuật biến OOP trong Java thành một yếu tố quan trọng trong lập trình.

1. Lập trình hướng đối tượng (OOP) trong Java là gì?

Lập trình hướng đối tượng trong Java là một phần quan trọng và có thể nói là không thể thiếu, nó được sử dụng để thiết kế một chương trình bằng cách sử dụng các lớp và đối tượng. Và được mô tả lại là cách chúng ta kiểm soát dữ liệu để truy cập mã. Với OOP, khi làm việc với dữ liệu, các lập trình viên phải xác định kiểu dữ liệu của một cấu trúc dữ liệu và các hoạt động được áp dụng cho cấu trúc dữ liệu đó. OOP trong Java luôn tập trung vào việc cải thiện khả năng đọc mã và khả năng sử dụng lại mã một cách hiệu quả.

Các nguyên tắc chính của lập trình hướng đối tượng bao gồm:

  • Sự trừu tượng (Abstraction)
  • Bao đóng (Encapsulation)
  • Kế thừa (Inheritance)
  • Đa hình (Polymorphism)

2. Đối tượng (Objects) là gì?

Đối tượng luôn được gọi là các thể hiện của một lớp (class) được tạo ra từ một lớp trong Java hoặc bất kỳ ngôn ngữ nào khác. Chúng có trạng thái (states) và hành vi (behaviour).

Ví dụ:

    public class Stringee {
        String company = "CONG TY CO PHAN STRINGEE";
    
        public static void main(String args[]) {
            Stringee stringee = new Stringee();
            System.out.println(stringee.company); // Kết quả: CONG TY CO PHAN STRINGEE
        }
    }

Chúng ta cũng có thể tạo nhiều đối tượng trong cùng một lớp và truy cập chúng trong một lớp khác:

    public class Stringee {
        String company = "CONG TY CO PHAN STRINGEE";
        String email = "info@stringee.com";
    }
    
    public class Show {
        public static void main(String[] args) {
            Stringee stringee1 = new Stringee();
            Stringee stringee2 = new Stringee();
            System.out.println(stringee1.company); // Kết quả: CONG TY CO PHAN STRINGEE
            System.out.println(stringee2.email);  // Kết quả: info@stringee.com
        }
    }

3. Lớp (Classes) là gì?

Lớp (Classes) có thể được coi như là các hàm khởi tạo (constructors) để tạo đối tượng. Tập hợp các đối tượng được gọi là một lớp. Lớp được xem là các đại lượng logic và không chiếm bất kỳ không gian nào trong bộ nhớ. Lớp còn được gọi là một bản mẫu (template) của một đối tượng.

Cú pháp khai báo một lớp:

    class tenLop {
        kieu bienThanhVien1;
        kieu bienThanhVien2;
        //...
        kieu bienThanhVienN;
    
        kieu tenPhuongThuc1(danhSachThamSo) {
            // thân phương thức
        }
    }

4. Trừu tượng (Abstraction) là gì?

Trừu tượng là quá trình hiển thị chỉ thông tin cần thiết và ẩn đi thông tin không cần thiết. Chúng ta có thể nói rằng mục đích chính của trừu tượng là ẩn dữ liệu. Trừu tượng giúp giảm độ phức tạp và công sức trong lập trình.

Ví dụ về lớp trừu tượng:

    abstract class Information {
        public abstract void company();
    }
    
    public class Stringee extends Information {
        public void company() {
            System.out.println("CONG TY CO PHAN STRINGEE");
        }
    
        public static void main(String args[]) {
            Information obj = new Stringee();
            obj.company(); // Kết quả: CONG TY CO PHAN STRINGEE
        }
    }

5. Kế thừa (Inheritance) là gì?

Kế thừa là một phương pháp trong đó một đối tượng kế thừa/tiếp nhận các thuộc tính của một đối tượng khác, đồng thời hỗ trợ phân loại theo cấp bậc.

Cú pháp:

    class A {
        // Nội dung của class A
    }
    
    class B extends A {
        // Nội dung của class B
    }

6. Đa hình (Polymorphism) là gì?

Đa hình ám chỉ nhiều hình thức, hoặc là một quá trình thực hiện một hành động duy nhất theo cách khác nhau. Đa hình chia thành hai loại:

  • Đa hình biên dịch (Compile-time polymorphism)
  • Đa hình runtime (Runtime polymorphism)

...

Ví dụ: Quản lý sinh viên sử dụng OOP

Code:

class Student {
    String name;
    int age;

    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public void displayInfo() {
        System.out.println("Name: " + name + ", Age: " + age);
    }
}

public class Main {
    public static void main(String[] args) {
        Student student = new Student("John", 20);
        student.displayInfo();
    }
}

Phân tích code

Trong ví dụ trên:

  • Class Student: Định nghĩa thuộc tính name và age, cùng với phương thức displayInfo.
  • Constructor: Khởi tạo đối tượng Student với tên và tuổi.
  • Object: Đối tượng student được tạo từ class Student.

Ý nghĩa của OOP

OOP giúp lập trình viên quản lý và tổ chức code tốt hơn, đặc biệt trong các dự án lớn. Nó cũng giúp tái sử dụng code và giảm thiểu lỗi bằng cách áp dụng các nguyên tắc rõ ràng.

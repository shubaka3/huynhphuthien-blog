3. Ví dụ câu hỏi phỏng vấn

Câu hỏi: "Hãy giải thích về Garbage Collection trong Java."

Garbage Collection (GC) trong Java là quá trình tự động dọn dẹp bộ nhớ bằng cách giải phóng bộ nhớ của các đối tượng không còn được tham chiếu nữa. Trong Java, bộ thu gom rác (GC) được quản lý bởi Java Virtual Machine (JVM) và không yêu cầu lập trình viên phải quản lý bộ nhớ thủ công. Các bước trong quá trình Garbage Collection thường bao gồm:

  • Mark: JVM đánh dấu tất cả các đối tượng mà chương trình còn tham chiếu.
  • Sweep: Các đối tượng không còn tham chiếu sẽ bị thu hồi.
  • Compact: Các đối tượng còn lại sẽ được di chuyển để giải phóng không gian bộ nhớ.

Garbage Collection giúp giảm thiểu nguy cơ rò rỉ bộ nhớ và nâng cao hiệu suất của ứng dụng.

Ví dụ về Garbage Collection trong mã Java:

Mã nguồn:

public class GarbageCollectionExample {
    public static void main(String[] args) {
        // Tạo đối tượng
        MyObject obj = new MyObject();
        
        // Đặt obj = null để không còn tham chiếu
        obj = null;

        // Gọi GC để kiểm tra việc dọn dẹp bộ nhớ (chỉ gọi khi JVM cần)
        System.gc();
    }

    static class MyObject {
        @Override
        protected void finalize() {
            System.out.println("Garbage Collection is called.");
        }
    }
}
